发现使用这个博客做记录还挺不错的，直接在 markdown 里写好，用#做区分，会自动在右侧做出摘要，可以点击跳转阅读，体验也不错，不用每天写一点就新建一篇文章了，毕竟也不是每天都有那么多事要写。

现在可以有点想法就在这里写下来，如果想法可以继续拓展，那就新建一篇文章拓展开，如果写的没啥价值，那就纯当记录了。

生活要有一个记录，不然这些过去的时光谁能记得呢？

关键在于去写

记得两年前就说要写博客了，但一直没写，原因呢？博客网站没做好～

其实好久之前就照着徐老师的 demo 把博客网站写得差不多了，但因为是一个完整的网站，包含了前后端和服务器，因此不好维护，出了一些问题后就跑去忙其他的了。

直到最近和朋友聊起这事时，朋友一句话点醒了我，博客最重要的不是去写吗？不用关注太多技术实现。

仔细一想确实是这样啊，自己在写博客这事上没有找到最重要的事，反而一直陷于技术选择上，浪费了太多时间。

于是迅速采用了 github page 和 hugo ，做了这个博客，使用起来非常简单，只需要在本地写好 markdown 文件，然后将编译好的文件推送到 github 上就可以了，而且还有相当不错的主题可以使用，做完整个博客也就半个小时，生成一篇新文章也就五个步骤。

学习的限度

不是所有的坑都有意义，很多坑都是毫无意义的，只会耽误你的时间。

新技术刚出来都是需要炮灰的，但你要想清楚，想不想自己当炮灰。

去关注最重要、最核心的事，这才是最有用的。

枚举的价值以及如何在 JS 中模拟枚举值—-> 序列与反序列化

枚举，一种从事先定义好的一组常量转换而来的遵从制约的变量

在拥有枚举的类型安全的语言中，枚举被理解为常量，也就是枚举成员以及它们的值都不会发生变化，而在 JS 中，虽然我们可能会重写一个变量为任何的类型，但我们有 Object.freeze() 方法可以将一个对象进行冻结，从而达到和常量一样的效果。

枚举的一个重要的作用是提高可读性

在 JS 这种弱类型的语言中模拟枚举最大的困难在于如何让枚举成员及枚举成员的值像强类型语言的常量一样不变。

有些博主是真的会怼人，哈哈

每天不写点手就会生

其实复利这个词不仅能用在金融上，在个人发展上也同样适用，如果你每天能多了解一点这世界，多学习一点东西，那么随着时间的推移，你的见识阅历自然就会大大增长。我对积累这事儿感受最深的还是高中时期，那时候天天上网，但因为手头紧，所以每次冲网费都是几块几块地冲，这真的是很小的钱，自己也没在意，直到有一天包夜无聊，点开自己的网吧账户查看，才突然发现自己累计消费已经三千多了！当时那叫一个震惊啊，每次几块几块地冲竟然会累积这么多，要知道这些钱都可以买一个自己的电脑了，也是从那之后，才真切体会到自己在网吧浪费了多长时间。尽管在网吧浪费时间这事自己是明确知道的，但知道一件事和一件事真切地对自己产生冲击还是不一样的。

钱具有复利性这点没错，但很多人其实不知道知识也是这样。我自己最深的体会就是，如果你关注了一个靠谱的公众号，一个靠谱的作者，那么大概率你以后会关注越来越多的好作者，看到更多的好文章，然后你的视野和见识也会越来越高，最终你会发现你和以前的自己差距越来越大。

所以很多时候我都觉得说一个人初心不改都是在骂人。一个人在他还小的时候，他对这个世界的认识肯定是有大量错误的，这时候许下的初心往往只是一厢情愿，是经不起时间的检验的，而一个人真的初心不改，那么有很大概率说明这个人的成长速度很慢，这么些年思想上并没有什么提高。

学习编程这事儿有时候真不能钻牛角尖，当一个概念你觉得太复杂一直搞不懂的时候，就不要再去钻了，这时候跳出来学看它最原始的定义，那就是真理，其他都是扯淡。如果一个概念太过复杂，那么它本身就是失败的，没必要太去钻。

在如今的世界，我们要学会有自己的可靠数据源。早在几年前我就意识到，媒体已经在变得不可信了，越来越多的媒体在传播真实的假事，即事情是真的，但我只说一部分。这是很取巧的一种方法，你看我说的都是真的吧，我可没有骗你哦，只是我只说了一半的真相，至于另一半，那不关我的事，反正我只关注流量，只在乎够不够刺激眼球。

所以，在如今的媒体中，可能没人在骗你，但你仍然被骗了。我们需要有自己的靠谱信息源，这个源可以是靠谱的人，靠谱的作者，靠谱的机构，但我们仍然要培养自己的分辨力，最终让自己有分辨真假的能力才是最重要的。

前端开发者如何选择自己的第二门语言

做为一名转行而来的前端开发，在熟悉前端之余肯定还想继续探索，首当其冲的就是学习第二门语言。

如今我们在第二门语言的学习上有了更多的选择，如 go、rust、python等，最诱人的当然是直接学习一门新语言，如go、rust，不仅更加强大，还能装一波逼～

但认真地想下，我们真的需要追逐潮流吗？

首先对于我来说，学一门新语言首先是为了增加自己对编程的认识，夯实基础。

那么学习新语言有必要吗？或者说 JAVA 之类的语言满足不了我的需求吗？

显然不是。首先语言的发展就有历史可循，我自己常说的就是如果想了解现在，最好先去了解历史，我们皆从过去而来。

那这样的话，我学习 JAVA 显然是更好的选择，强类型，面向对象，历史悠久，有足够的文档支持，显然这会是我更好的选择，虽然学习 go 和 rust 会让我显得更酷～
