关键在于去写

记得两年前就说要写博客了，但一直没写，原因呢？博客网站没做好～

其实好久之前就照着徐老师的 demo 把博客网站写得差不多了，但因为是一个完整的网站，包含了前后端和服务器，因此不好维护，出了一些问题后就跑去忙其他的了。

直到最近和朋友聊起这事时，朋友一句话点醒了我，博客最重要的不是去写吗？不用关注太多技术实现。

仔细一想确实是这样啊，自己在写博客这事上没有找到最重要的事，反而一直陷于技术选择上，浪费了太多时间。

于是迅速采用了 github page 和 hugo ，做了这个博客，使用起来非常简单，只需要在本地写好 markdown 文件，然后将编译好的文件推送到 github 上就可以了，而且还有相当不错的主题可以使用，做完整个博客也就半个小时，生成一篇新文章也就五个步骤。

学习的限度

不是所有的坑都有意义，很多坑都是毫无意义的，只会耽误你的时间。

新技术刚出来都是需要炮灰的，但你要想清楚，想不想自己当炮灰。

去关注最重要、最核心的事，这才是最有用的。
