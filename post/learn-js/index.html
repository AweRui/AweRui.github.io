为什么要关注它的整体？

上来不要陷入技术细节，先搞清楚它的历史，它的定位，它解决了什么问题，它自己又有什么问题

学一门语言或技术，最好是先对它有个整体的了解，这样才不会走弯路，才知道自己努力的方向对不对

过早地陷入技术细节，只会让自己脑子一片混乱，学不到关键

我们都觉得 this 比较乱，那么老问题来了，为什么要这么设计呢？

原因和 JS 里面对象的数据结构有关，因为在 JS 里一切皆对象，所以函数经常是放在一个对象里，作为对象属性的值来存储的，并且这个值是一个地址，在使用时会先查找到这个对象的地址，再从这个地址里面找到原始的对象并返回这个属性的值，而函数是作为属性的值来保存的，在保存时就是一个地址，所以函数可以在不同的上下文去执行，但不同上下文有不同的环境，你的 name 和我的 name 不一样，那如何来表示这个东西呢，这时就需要 this 了，它的设计目的就是在函数体的内部去指代函数被调用时的环境，以此来实现在不同的环境中去处理同一个表达
